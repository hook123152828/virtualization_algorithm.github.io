演算法原理
使用 splitmix64 做 key 的混淆，避免 unordered_map 被針對性測資 hack，保持 O(1) 均攤效率。
Hashsplitmix64抗Hack
虛擬碼
FIXED_RANDOM = random_seed
splitmix64(x):
x += 0x9e3779b97f4a7c15
x = (x ^ x>>30) * 0xbf58476d1ce4e5b9
x = (x ^ x>>27) * 0x94d049bb133111eb
return x ^ x>>31
hash(key) = splitmix64(key ^ FIXED_RANDOM)
